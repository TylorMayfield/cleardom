# ClearDOM Scenario Fixtures

These fixtures are intentionally small demo codebases for end-to-end scanner testing.
They complement the WCAG benchmark by modeling common project shapes:

- `next-app`: Next.js App Router files with route exclusions and web accessibility issues.
- `react-design-system`: React app code that relies on custom component mappings.
- `react-native-app`: React Native screen code with native accessibility props.
- `monorepo`: Multiple package roots, ignored build output, and include/exclude behavior.
- `runtime-demo`: Static HTML for manual/runtime scanner demonstrations.
